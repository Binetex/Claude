# Changelog — Floremart

Ветка `refactor/agent-architecture-foundation` (задеплоена на floremart.com). Более ранняя
история — в `docs/HANDOFF.md`.

## 2026-07-21

### QUO (ex-OpenPhone) — коммуникации
- Полная интеграция: входящие/исходящие SMS, звонки (отвеченные/пропущенные/voicemail), записи,
  транскрипты, summary; авто-привязка к заказам по нормализованному телефону; unread; индикаторы в
  списке заказов; страница «Нераспознанные» с ручной привязкой/игнором; 30-дневный backfill (CLI).
- Подключён и активирован live для магазина **TheFlow** (номер `+13238008421`).
- Fix: исходящие звонки больше не отбрасываются как self-call (определение собеседника через
  `direction`, `participants=[from,to]`).
- Fix: URL записи догружается одним GET, если `call.recording.completed` пришёл без него.
- Известное: транскрипт из приложения QUO недоступен через публичный API (Sona/тариф); пропущенный
  звонок без номера участника не привязывается (номер не выдумываем).

### WooCommerce
- Авто-назначение основного флориста для оплаченных заказов (паритет с Shopify).
- Отображение `failed`-оплаты как «Ошибка оплаты» (UI-флаг, без отдельного статуса).
- Перенос billing-адреса в поля отправителя (`senderAddress*`).
- **Fees → чаевые (`tip`)**: раньше `tip` был всегда 0.

### Burq (доставка)
- Per-site «Default Burq dropoff instructions» (`Site.burqDefaultDropoffInstructions`, additive
  migration) — единый текст, добавляемый в `dropoff.notes` новых draft.
- В карточке заказа: «Курьер вызван» / «Доставка завершена» (местное время магазина); убраны
  несинхронизируемые «Статус: Ожидает» и «Готовность».
- POD-фото открывается в лайтбоксе внутри дашборда, без скачивания.

### Прибыль
- `estimatedProfit` теперь включает чаевые: `itemsTotal − floristTotal − deliveryActualCost + tip`
  (налог — сквозной).

### Операции с данными (прод, разово)
- Tip-backfill: 67 Woo-заказов (Fees → tip), только `Order.tip`.
- Recompute прибыли: 52 заказа (гард `floristTotal>0` + PAID/PAY_LATER_APPROVED + не CANCELLED).
- Удалены 2 временных тест-заказа `#QUO-TEST-*`.
- Откат инцидента: `syncWooOrders(fullHistory)` создал 155 старых заказов (2024) и массово назначил
  их — все откачены с гардом (Orders/Assignments/Deliveries вернулись к прежним значениям).
  **Урок:** `fullHistory`-sync тянет всю историю магазина + триггерит авто-назначение/доставку —
  для точечных правок использовать прямые скрипты только на нужные поля, не ingest.
